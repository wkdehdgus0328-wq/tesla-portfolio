## 학습 기록: 폐쇄형 MCP 서버 구축

### 1. 프로젝트 목표

- **기존 한계:** 엑셀 VBA 방식은 개인 PC 수준의 자동화에 머물며, 확장성과 보안성에 한계가 존재.
- **개선 방향:** 사내 데이터를 보호할 수 있는 **폐쇄형(Private) 서버 환경**을 구축하고, AI가 스스로 도구(Tool)를 사용하여 업무를 수행하는 **에이전틱 워크플로우(Agentic Workflow)** 구현.

### 2. 핵심 도입 기술 (Tech Stack)

| **구분** | **도입 기술** | **역할 및 선정 이유** |
| --- | --- | --- |
| **연결 표준** | **MCP (Model Context Protocol)** | AI 모델과 사내 데이터(엑셀, 파일 등)를 연결하는 표준 규약. 모델 변경 시 유연성 확보 및 연결 구조 표준화. |
| **운영 도구** | **Gemini CLI** | 터미널 기반의 AI 제어 인터페이스. GUI 없이 명령어로 대규모 작업을 자동화하고 시스템을 통제하는 조종간 역할. |
| **실행 환경** | **Node.js Server** | 자바스크립트 기반의 서버 환경. 엑셀 API와 AI 사이에서 데이터를 중계하고 처리하는 백엔드 엔진. |

---

### 2. 단계별 구축 과정 (Process Pipeline)

### [1단계] AI 제어 환경 구축 (Gemini CLI 도입)

가장 먼저 AI를 웹이 아닌 **시스템 터미널(명령어 환경)**로 불러오는 작업을 수행했습니다.

- **실행 내용:**
    - 개발자 도구(Node.js 등)를 활용하여 로컬 PC에 **Gemini CLI 설치**.
    - Google Cloud 인증 및 API Key 연동을 통해 터미널 제어 권한 획득.
- **구축 의의:**
    - 마우스 클릭 없이 명령어로 대규모 작업을 처리할 수 있는 **'AI 조종간(Control Tower)'** 확보.
    - 단순 대화형 AI가 아닌, 시스템 명령을 수행하는 **개발자 레벨의 접근성** 확보.

### [2단계] 사내 AI 서버 자원 확보 (MCP Server Clone)

AI가 실제로 데이터를 만질 수 있도록, 사내망 역할을 하는 로컬 서버를 준비했습니다.

- **실행 내용:**
    - GitHub를 통해 검증된 오픈소스 MCP 서버(`excel-mcp-server`)를 **Git Clone**하여 로컬 환경에 복제.
    - `npm install`을 통해 서버 구동에 필요한 라이브러리(Dependencies) 일괄 설치.
- **구축 의의:**
    - 외부 클라우드 서비스에 의존하지 않고, **내 컴퓨터(사내망) 안에서만 돌아가는 독립적인 서버** 마련.
    - **폐쇄형(On-Premise) LLM 환경**의 기초 인프라 완성.

### [3단계] AI와 서버의 신경망 연결 (Configuration)

설치한 '두뇌(Gemini CLI)'와 '신체(MCP Server)'를 서로 연결하는 설정 작업을 진행했습니다.

- **실행 내용:**
    - CLI 설정 파일(`settings.json`)에 로컬 서버 경로와 실행 명령어를 등록.
    - 서버를 구동(`npm start`)하고, CLI에서 `/mcp` 명령어로 **연결 상태(Connected) 및 통신 보안** 점검.
- **구축 의의:**
    - **MCP 프로토콜**을 적용하여, 향후 AI 모델이 바뀌더라도 데이터 연결 구조는 그대로 유지되는 **표준화된 연결 체계** 수립.

### [4단계] 도구(Tool) 확장 및 자율 에이전트 구동

AI에게 단순 텍스트 생성이 아닌, 실제 업무를 수행할 수 있는 '권한(Tool)'을 부여하고 검증했습니다.

- **실행 내용:**
    - AI에게 **"디렉토리 구조 파악"** 및 **"엑셀 데이터 읽기/쓰기"** 툴 사용 권한 부여.
    - 실제 엑셀 파일 생성을 명령하여, AI가 스스로 파일을 만들고 데이터를 채워 넣는 **Agentic Workflow(에이전트 워크플로우)** 시연 성공.
- **구축 의의:**
    - 사람이 엑셀을 열지 않아도 AI가 백그라운드에서 데이터를 처리하는 **완전 무인 자동화(Full Automation)** 구현.
    - 문서 분석, 데이터 입력, 보고서 작성을 스스로 수행하는 **'행동하는 AI'** 검증 완료.

### **[Agentic Workflow] 자연어 기반 화장품 CRM 데이터 분석 및 전략 자동화**

- **Role:** AI 서비스 기획 & 워크플로우 설계
- **내용:**
    - **MCP(Model Context Protocol) 서버**를 활용하여 LLM이 로컬 엑셀 파일을 직접 제어(Read/Write)하는 환경 구축.
    - 자연어 프롬프트를 통해 고객 데이터(100명)의 **RFM 분석 및 세그먼테이션(VIP, 이탈위험 등) 자동화**.
    - Python 스크립트 생성을 유도하여 **고객 분포 및 만족도 차트(Graph) 생성** 및 시각화.
    - 데이터 분석 결과에 기반한 **타겟 그룹별 맞춤형 CRM 마케팅 전략(Action Plan)** 자동 생성 구현.
- **성과:** 반복적인 데이터 핸들링 시간을 90% 이상 단축하고, 비개발 직군도 고도화된 CRM 분석이 가능한 에이전틱 워크플로우(Agentic Workflow) 모델 제시.
