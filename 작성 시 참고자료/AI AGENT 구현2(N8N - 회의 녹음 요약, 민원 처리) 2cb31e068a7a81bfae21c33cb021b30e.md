# AI AGENT 회의 녹음 요약하기 - N8N

### 1. 프로젝트 개요

- **목표:** 녹음된 음성 파일(.m4a 등)을 업로드하면 AI가 자동으로 텍스트로 변환하고 요약하여 로컬에 저장 및 응답.
- **사용 툴:** n8n (Self-hosted), OpenAI API (Whisper, GPT-4o 등)
- **핵심 기능:** STT(Speech-to-Text) 및 LLM 요약 자동화

### 2. 워크플로우 흐름 (Node Flow)

n8n 워크플로우는 총 6단계로 구성됨.

1. **Webhook (POST):**
    - 역할: 로컬 HTML 페이지에서 전송한 음성 파일을 수신.
    - 설정: Method `POST`, Path `upload`, `multipart/form-data` 지원.
2. **Transcribe a recording (OpenAI):**
    - 역할: 수신한 오디오 파일(Binary)을 텍스트로 변환 (STT).
    - 모델: Whisper.
3. **AI Agent (with OpenAI Chat Model):**
    - 역할: 변환된 텍스트를 입력받아 핵심 내용 요약 수행.
    - 프롬프트: "음성 메모 핵심 내용 불렛 포인트로 요약해줘" 등.
4. **Convert to File:**
    - 역할: AI가 요약한 텍스트 데이터를 파일(File) 형태로 변환.
5. **Read/Write Files from Disk:**
    - 역할: 요약된 텍스트 파일을 컴퓨터(서버) 로컬 경로에 저장.
6. **Respond to Webhook:**
    - 역할: 업로드를 요청한 웹페이지(HTML)에 작업 완료 결과(또는 요약문)를 반환.

### 프로젝트 개요: AI 기반 민원 데이터 자동 분석 및 퍼스널 이메일 응답 시스템

- **목적**: 엑셀 등 대규모 민원 파일(약 40개 열)을 AI가 자동 분석하여 핵심 안건을 분류하고, 각 민원인에게 맞춤형 디자인이 적용된 HTML 보고서를 자동 발송함으로써 민원 처리 효율성을 극대화함.
- **핵심 도구**:
    - **n8n**: 전체 워크플로우 자동화 및 데이터 오케스트레이션.
    - **OpenAI GPT-4o**: 민원 내용 분석, 카테고리 분류, 인사이트 도출.
    - **HTML/CSS**: 수신자별 이름이 적용된 맞춤형 이메일 디자인 및 브랜드 이미지(레제) 통합.
- **주요 성과**: 수작업으로 진행되던 민원 분류 및 개별 응답 프로세스를 자동화하여 업무 시간을 90% 이상 단축하고, 일관된 비즈니스 톤앤매너 유지.

### 워크플로우 흐름 (Workflow Step-by-Step)

1. **데이터 입력 (Extract from File)**
    - 접수된 민원 파일(XLSX, CSV 등)에서 '제목', '내용', '민원인 이름' 등 약 40여 개의 데이터 열을 추출합니다.
2. **데이터 결합 및 구조화 (Merge)**
    - 원본 데이터와 이후 생성될 AI 분석 데이터를 정확히 매핑하기 위해 위치(Position) 기반의 Merge 노드를 사용하여 데이터 레코드를 단일 객체로 결합합니다.
3. **AI 비즈니스 인텔리전스 분석 (AI Agent)**
    - **전략적 페르소나**: '20년 경력의 전문 행정관' 페르소나를 부여하여 민원의 본질적 이슈를 파악합니다.
    - **구조화된 추출**: 단순 요약을 넘어 [분류, 핵심 요약, 근본 이슈/인사이트, 실행 방안]을 시스템 처리가 용이한 구조로 도출합니다.
4. **조건부 라우팅 (Switch)**
    - AI가 분류한 카테고리(예: 감사, 행정, 복지 등)를 식별하여 부서별 또는 중요도별로 처리 경로를 자동 분기합니다.
5. **개인화 HTML 이메일 발송 (Send Email)**
    - **동적 변수 적용**: `{{ $json["이름"] }}` 변수를 활용하여 수신자별 이름을 이메일 본문에 자동 삽입합니다.
    - **비주얼 리포트 구성**: 정중한 거절이나 안내 메시지에 브랜드 이미지(레제)를 결합한 고해상도 HTML 템플릿을 사용하여 전문적인 인상을 전달합니다.
6. **처리 완료 및 응답 (Respond to Webhook)**
    - 모든 프로세스가 성공적으로 완료되었음을 시스템에 알리고 최종 데이터를 확정합니다.
